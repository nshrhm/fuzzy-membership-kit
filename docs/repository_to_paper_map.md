# Repository-to-Paper Map

This document maps Paper 1 manuscript elements to artifacts in the `fuzzy-membership-kit` hub repository.  It is intended to help authors keep claims traceable and help reviewers find the relevant implementation, tests, and reproducibility metadata.  It is not a manuscript and does not contain paper-specific datasets.

| Paper section | Claim / definition / result | Repository artifact | Tests or validation | Reproduction output | Reviewer-facing check |
| --- | --- | --- | --- | --- | --- |
| Classical membership functions | Baseline triangular, trapezoidal, Gaussian, S/Z/pi, and sigmoid-composed S/Z/pi functions | `src/fuzzymf/core.py`; `docs/theory.md`; `docs/theory_ja.md` | `tests/test_core.py`; `tests/test_io.py` for config-backed use | Existing examples under `examples/configs/`; optional plots from `fuzzymf-plot` | Check definitions, domain/range behavior, continuity notes, and expected shapes. |
| Focus-aware warping framework | Composition formula `mu(u)=h(w_theta(u))` | `src/fuzzymf/focus.py`; `docs/warping_framework.md`; `docs/warping_framework_ja.md` | `tests/test_focus.py` | Demonstrated through `scripts/reproduce_paper1_diagnostics.py` membership rows | Check composition behavior and the distinction between `w(far)=q` and `mu(far)=h(q)`. |
| Warping families | Logistic, tanh, arctan, Gompertz, and generalized logistic warps | `src/fuzzymf/warping.py` | `tests/test_warping.py` | Warp rows in the Paper 1 diagnostics CSV | Check `w(focus)=0.5`, `w(far)=q`, monotonicity, finite values, and range. |
| Anchor solving | Semantic-anchor parameter determination from `focus`, `far`, and warped-coordinate target `q` | `src/fuzzymf/anchors.py` | `tests/test_anchors.py` | Parameters are used by the warp diagnostics script | Check that parameters are determined from declared semantic anchors rather than arbitrary tuning. |
| Diagnostics | Range, monotonicity, local discriminability, tail compression, slope, curvature, and anchor error | `src/fuzzymf/diagnostics.py`; `docs/diagnostics.md`; `docs/diagnostics_ja.md` | `tests/test_diagnostics.py` | Diagnostics CSV from `scripts/reproduce_paper1_diagnostics.py` | Check that metrics correspond to stated design goals and distinguish warp targets from membership targets. |
| Paper 1 reproduction example | Warping-family comparison and diagnostic demonstration with `focus=50`, `far=210`, `q=0.9` | `examples/configs/paper1_warping_comparison.yaml`; `scripts/reproduce_paper1_diagnostics.py` | Smoke run of the script; standard pytest/ruff checks | CSV under `results/csv/` when run with default output; optional figure under `results/figures/` if matplotlib is installed | Check that CSV can be regenerated without external data and includes both `warp` and `membership` rows. |
| Reproducibility metadata | Artifact availability, package version, and file-level checksums | `MANIFEST.json`; `scripts/generate_manifest.py`; `pyproject.toml`; `CITATION.cff` | Manifest generator plus checksum inspection; package checks | Commit-friendly `MANIFEST.json` excludes transient git state by default | Check package version, dependency metadata, file checksums, and citation metadata. |
| Documentation separation | Theory, implementation, diagnostics, and reviewer notes are separated from future paper manuscripts | `docs/theory*.md`; `docs/implementation*.md`; `docs/warping_framework*.md`; `docs/diagnostics*.md`; `docs/reviewer_notes.md` | Documentation consistency by inspection; optional MkDocs build | MkDocs site generated locally by `mkdocs build` | Check that equations, code usage, and reproducibility claims are traceable without adding a manuscript to this repository. |
