# フォーカス-aware ワーピング枠組み

このページは Paper 1 の v0.2.0.dev0 ドラフト基盤を説明します。仮題は「Focus-Aware Membership Functions via Universe-of-Discourse Warping: A Comparative and Reproducible Framework」です。

中心となる表現は次の式です。

\[
\mu(u) = h(w_\theta(u))
\]

ここで `u` は議論領域上の値、`w_theta` はワーピング関数、`h` は S、Z、pi などの基底メンバーシップ関数です。ワーピングは元の議論領域を `[0, 1]` に写し、その後で基底メンバーシップを評価します。

## 意味的アンカー

このドラフトでは、ワーピング関数 `w` に対して 2 つのアンカーを明示します。

- `focus` (`u*`): 意味的な焦点。`w(focus)=0.5` を満たします。
- `far` (`uc`): ワープ後座標での遠方アンカー。`w(far)=q` を満たします。
- `q`: 遠方アンカーのワープ後座標での目標値で、`(0.5, 1)` の範囲に置きます。

合成メンバーシップ関数 `mu(u)=h(w(u))` では、これらは次を意味します。

\[
\mu(focus)=h(0.5),\qquad \mu(far)=h(q)
\]

したがって、`q` は必ずしも最終的なメンバーシップ値ではありません。たとえば `h(x)=S(x; 0.1, 0.9)` かつ `q=0.9` の場合、`h(q)=1.0` なので、遠方点でのメンバーシップ目標は 1.0 です。

`far > focus` のときワーピングは増加、`far < focus` のときワーピングは減少になります。同じアンカー規約で両方向の概念を扱えます。

## ワーピング族

パッケージは logistic、tanh、arctan、Gompertz、generalized logistic のワーピングを提供します。各族には閉形式のアンカー解法があり、ブラックボックスのフィッティングではなく、意味的アンカーとしてパラメータを報告できます。既存の sigmoid-composed S 関数は、この広い枠組みの特殊例として、logistic ワーピングの後に S 関数を適用したものと見なせます。

## 解釈

この枠組みは、焦点値の近くでの局所的な識別性と、遠方領域での滑らかな圧縮を明示的に制御します。ただし、どの概念やデータセットにも普遍的に最良のメンバーシップ関数を主張するものではありません。
